# 贡献指南

感谢你有兴趣为这个合集贡献！这个指南说明怎么提交新的 skill 条目。

## 收录标准

收录的 skill 需要满足：

1. **真实可用** - 你自己安装过、用过，确认它能工作。
2. **有公开仓库** - GitHub 公开仓库，有 README 说明。
3. **维护活跃** - 最近 6 个月内有提交。
4. **文档清晰** - README 说清楚做什么、怎么安装、怎么用。
5. **不含恶意代码** - 不窃取数据、不自动执行危险操作。

## 怎么提交

### 提交新条目

1. Fork 这个仓库。
2. 编辑 README.md，在对应分类下加一行，格式：

```markdown
- [项目名](https://github.com/用户名/仓库名#readme) - 一句话客观描述它做什么。
```

3. 如果现有分类都不合适，可以在末尾提议新分类。
4. 提交 PR，标题用 `Add: 项目名`。

### 条目格式要求

- 链接以 `#readme` 结尾。
- 描述大写开头（英文）或正常句式（中文），句号结尾。
- 不用"最""神器""amazing""best"这类词。
- 描述说清楚它**做什么**，不说它**多厉害**。

### 好的描述

- ✅ `文章写作辅助 skill，从素材整理到初稿质检的全流程。`
- ✅ `C++ 信息学奥赛题解生成器，给题目链接自动生成带注释的参考代码。`

### 不好的描述

- ❌ `史上最强的写作神器！`
- ❌ `一个 amazing 的工具`
- ❌ `非常好用的 skill`

## 审核流程

- PR 提交后会在 7 天内审核。
- 审核会确认：条目格式、链接可访问性、收录标准。
- 通过后合并到主分支。

## 问题反馈

发现某条 skill 已失效、链接打不开、或分类有误，直接提 Issue。
